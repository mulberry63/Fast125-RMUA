STEP 01 检查 & 安装 NVIDIA 驱动

仿真器基于 UE4，必须有 NVIDIA 独立显卡。首先确认驱动已正确安装，建议驱动版本 ≥ 470。

检查当前驱动状态
bash 
# 查看 GPU 和驱动版本
nvidia-smi

# 如果命令不存在，说明驱动未安装
nvidia-smi --query-gpu=name,driver_version --format=csv
如果驱动未安装 — 安装推荐驱动
bash 
# 方法一：使用 ubuntu-drivers（推荐新手）
sudo apt update
sudo ubuntu-drivers autoinstall

# 重启后验证
sudo reboot
nvidia-smi

# 方法二：手动指定版本（如需 525 驱动）
sudo apt install -y nvidia-driver-525
⚠️ 虚拟机用户注意：如果你在 VMware / VirtualBox 里运行 Ubuntu，仿真器无法正常运行（UE4 需要直通 GPU）。请确保是物理机或者双系统安装。
STEP 02 安装 ROS Noetic

比赛使用 ROS 1 Noetic（Ubuntu 20.04 对应版本），这是与仿真器和开发模板通信的核心框架。

添加 ROS 源 & 安装
bash · 完整安装（推荐） 
# 1. 添加 ROS 软件源
sudo sh -c 'echo "deb http://packages.ros.org/ros/ubuntu $(lsb_release -sc) main" \
  > /etc/apt/sources.list.d/ros-latest.list'

# 2. 添加 GPG 密钥
sudo apt install -y curl
curl -s https://raw.githubusercontent.com/ros/rosdistro/master/ros.asc | \
  sudo apt-key add -

# 3. 更新并安装完整版 ROS（包含所有常用工具）
sudo apt update
sudo apt install -y ros-noetic-desktop-full

# 4. 安装依赖管理工具
sudo apt install -y python3-rosdep python3-rosinstall \
  python3-rosinstall-generator python3-wstool build-essential

# 5. 初始化 rosdep
sudo rosdep init
rosdep update
配置环境变量（每次开终端自动生效）
bash 
# 写入 .bashrc，永久生效
echo "source /opt/ros/noetic/setup.bash" >> ~/.bashrc
source ~/.bashrc

# 验证安装
rosversion -d
# 应输出: noetic
创建 catkin 工作空间
bash 
mkdir -p ~/catkin_ws/src
cd ~/catkin_ws
catkin_make
echo "source ~/catkin_ws/devel/setup.bash" >> ~/.bashrc
source ~/.bashrc
💡 小科普：catkin_ws 就是你的 ROS 项目工作目录，你以后写的所有 ROS 包都放在 ~/catkin_ws/src/ 里，用 catkin_make 编译。
STEP 03 安装 Docker

官方要求最终提交的算法打包成 Docker 镜像。先在主机开发，后期封装镜像提交。

安装 Docker Engine
bash · 官方源安装 
# 1. 安装前置依赖
sudo apt-get install -y ca-certificates gnupg lsb-release

# 2. 添加 Docker 官方 GPG 密钥
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | \
  sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg

# 3. 添加 Docker 软件源
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] \
  https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | \
  sudo tee /etc/apt/sources.list.d/docker.list > /dev/null

# 4. 安装 Docker
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 5. 把当前用户加入 docker 组（避免每次都要 sudo）
sudo usermod -aG docker $USER
newgrp docker

# 6. 验证安装
docker run hello-world
⚠️ 执行 newgrp docker 后需要重新登录或重启终端，组权限才能完全生效。
STEP 04 安装 nvidia-container-toolkit

让 Docker 容器能访问宿主机的 NVIDIA GPU，这是运行仿真器 Docker 版本的关键依赖。

安装 toolkit
bash 
# 1. 添加 nvidia-docker 源
distribution=$(. /etc/os-release; echo $ID$VERSION_ID)

curl -s -L https://nvidia.github.io/nvidia-docker/gpgkey | sudo apt-key add -
curl -s -L https://nvidia.github.io/nvidia-docker/$distribution/nvidia-docker.list | \
  sudo tee /etc/apt/sources.list.d/nvidia-docker.list

# 2. 安装
sudo apt-get update && sudo apt-get install -y nvidia-container-toolkit

# 3. 重启 Docker daemon
sudo systemctl restart docker

# 4. 验证：在 docker 中调用 nvidia-smi
sudo docker run --rm --runtime=nvidia --gpus all ubuntu nvidia-smi
# 正常输出应显示你的 GPU 信息
✅ 如果 docker run ... nvidia-smi 正常输出 GPU 信息（显卡名称、驱动版本），说明 GPU 直通 Docker 配置成功。
STEP 05 下载仿真器

从官方阿里云 OSS 下载 RMUA 2026 仿真器包，并配置好启动所需的 settings.json。

下载并解压仿真器
bash 
# 创建目录
mkdir -p ~/rmua2026 && cd ~/rmua2026

# 下载仿真器（RMUA 2026 官方包）
wget https://sz-rm-rmua-dispatch-prod.oss-cn-shenzhen.aliyuncs.com/0e954d75cf7e0afdba0d2dea27a08295/simulator_12.0.0.3.zip

# 解压
unzip simulator_12.0.0.3.zip
cd simulator_12.0.0.3
ls -la
克隆 Simulator 仓库（获取 settings.json 和启动脚本）
bash 
cd ~/rmua2026
git clone -b RMUA2026-01 \
  https://github.com/RoboMaster/IntelligentUAVChampionshipSimulator.git

# 将 settings.json 复制到 AirSim 配置目录
mkdir -p ~/Documents/AirSim
cp IntelligentUAVChampionshipSimulator/settings.json ~/Documents/AirSim/settings.json
给启动脚本加执行权限
bash 
cd ~/rmua2026/simulator_12.0.0.3
chmod +x *.sh
ls *.sh
# 应该能看到：run_simulator.sh  run_simulator_offscreen.sh 等
💡 性能提示：如果帧率不稳定，可以编辑 ~/Documents/AirSim/settings.json，注释掉不需要的相机配置（比如后视相机）来降低 GPU 负担。
STEP 06 克隆 Base 开发环境

官方提供了开发模板仓库 IntelligentUAVChampionshipBase，里面包含 ROS 包示例、Docker 模板和接口定义，是你们写代码的起点。

克隆仓库
bash 
cd ~/rmua2026
git clone -b RMUA2026 \
  https://github.com/RoboMaster/IntelligentUAVChampionshipBase.git

ls IntelligentUAVChampionshipBase/
# 主要目录：
# basic_dev/          ← 开发模板（包含 airsim_ros 包 + Dockerfile）
# VelCmdmsg/          ← 速度控制消息定义
# README.md
将 airsim_ros 包加入工作空间
bash 
# 把 airsim_ros 包链接/复制到 catkin_ws
cp -r ~/rmua2026/IntelligentUAVChampionshipBase/basic_dev/airsim_ros \
  ~/catkin_ws/src/

# 安装依赖
cd ~/catkin_ws
rosdep install --from-paths src --ignore-src -r -y

# 编译
catkin_make
source devel/setup.bash
构建 basic_dev Docker 镜像（可选，用于提交测试）
bash 
cd ~/rmua2026/IntelligentUAVChampionshipBase/basic_dev
docker build -t basic_dev .
# 构建可能需要 10~20 分钟（需要下载 ROS 等依赖）

STEP 08 了解关键 ROS 话题接口

这是你们写控制代码必须知道的话题列表，分为输入（传感器数据）和输出（控制指令）两类。

📥 输入话题（仿真器 → 你的程序）
话题名类型频率说明
/airsim_node/drone_1/odom_local_ned Odometry 10Hz 初始化位姿（世界系）
/airsim_node/local_goal PoseStamped 10Hz 当前路径终点坐标
/airsim_node/drone_1/imu/imu Imu 100Hz IMU 加速度/角速度
/airsim_node/drone_1/gps/gps NavSatFix 10Hz GPS（工厂内为 0）
/airsim_node/drone_1/front_left_custom/Scene Image 20Hz 前视左目相机图像
/airsim_node/drone_1/front_right_custom/Scene Image 20Hz 前视右目相机图像
/airsim_node/drone_1/lidar/LidarCustom PointCloud2 10Hz MID360 激光雷达点云
/airsim_node/drone_1/wind_sensor Vector3Stamped 50Hz 风速计（世界系，m/s）
📤 输出话题（你的程序 → 仿真器，二选一）
话题名字段说明最大频率
/airsim_node/drone_1/vel_cmd_body_frame x速度 / y速度 / z速度 / 角速度 / 加速度上限(≤8m/s²) / 急停标志 100Hz
/airsim_node/drone_1/rotor_pwm_cmd 四电机 PWM（0:右前 / 1:左后 / 2:左前 / 3:右后） 100Hz
/airsim_node/meter_report index（工厂编号）/ value（仪表数值 0~50） —
💡 新手建议：先用 VEL 速度控制（vel_cmd_body_frame），比 PWM 简单得多。发一个 x=1.0 的速度指令，无人机就会向前飞。两种接口不能混用，以第一次接收到的信号类型为准。
Hello World：发一条速度指令让无人机动起来
bash · 手动测试（仿真器运行时执行） 
# 先 source 你的工作空间
source ~/catkin_ws/devel/setup.bash

# 发布一次速度指令（向前飞 1m/s，持续手动 ctrl+C 停止）
# 字段顺序: [vx, vy, vz, yaw_rate, acc_max, is_stop]
rostopic pub -r 10 /airsim_node/drone_1/vel_cmd_body_frame \
  rmua_airsim_ros_pkgs/VelCmd \
  "twist: {linear: {x: 1.0, y: 0.0, z: 0.0}, angular: {z: 0.0}}"
下一步 环境OK之后做什么？

配置完成后，按以下顺序开始算法开发：

学习路线 
阶段 1 → 飞行控制基础
  ├─ 订阅 odom + local_goal，实现飞向目标点的 PID 控制
  └─ 验证：无人机能自主飞到指定坐标

阶段 2 → SLAM 定位
  ├─ 接入 FAST-LIO2（LiDAR + IMU 融合）
  └─ 验证：可获得比 GPS 更精确的实时位姿

阶段 3 → 障碍物避障
  ├─ 点云过滤 + 碰撞检测
  └─ 验证：遇到悬浮汽车能自动绕行

阶段 4 → 工厂巡检视觉
  ├─ 识别红色三灯设备（YOLOv8 / OpenCV HSV）
  ├─ 仪表盘读数（指针角度 → 数值）
  └─ 通过 /airsim_node/meter_report 上报

阶段 5 → 系统整合 + Docker 封装
  └─ 完整流程测试 → 打包 Docker → 首次提交
